用户反馈：还是一样，进度条不更新，完成状态也不更新，要刷新页面才会更新

执行内容：
**根本原因**：
后端发送SSE事件时没有指定 `event:` 字段，导致所有事件都被当作默认的 `message` 事件。前端监听的是特定事件类型（task_update、task_added等），所以无法接收到事件。

**问题定位**：
- 后端（server.ts:1963）：`res.write(\`data: ${JSON.stringify(data)}\\n\\n\`)`
- 前端（app.js:2391-2428）：监听 task_update、task_added、task_removed、queue_status 事件
- 事件类型不匹配，导致前端无法接收

**修复方案** (server.ts:1963-1965)：
在发送SSE数据时，根据 data.type 设置正确的 event 字段：
```typescript
res.write(`event: ${data.type}\n`)
res.write(`data: ${JSON.stringify(data)}\n\n`)
```

修复效果：
- SSE事件能够正确发送和接收
- 进度条实时更新
- 任务状态实时更新
- 无需刷新页面

1. **添加方式**：支持上传脚本文件 + 输入URL下载两种方式
2. **客户端模式**：智能路由（服务器根据歌曲信息自动选择最佳源）
3. **支持操作**：musicUrl（获取播放链接）、search（搜索音乐）、lyric（获取歌词）、pic（获取封面）
4. **权限管理**：不需要（所有用户共享所有自定义源）

## 现有系统架构（来自探索结果）

### 自定义源系统
- **核心文件**：`src/server/userApi.ts`
- **脚本存储**：`data/data/users/source/_open/` 和 `data/data/users/source/{username}/`
- **配置文件**：`sources.json` 存储源元数据
- **沙箱环境**：VM2 虚拟机，10秒超时
- **API接口**：
  - `loadUserApi(scriptPath, owner)` - 加载单个脚本
  - `initUserApis()` - 初始化所有脚本
  - `callUserApiGetMusicUrl(source, songInfo, quality)` - 获取音乐URL
  - `isSourceSupported(source, clientUsername)` - 检查源是否支持
  - `getLoadedApis()` - 获取所有已加载的源

### 脚本 API
脚本通过 `lx` 对象与服务器通信：
- `lx.send('inited', { sources: {...} })` - 注册源
- `lx.on('request', handler)` - 处理请求
- `lx.request(url, options, callback)` - HTTP 请求

### 支持的 Actions
- `musicUrl` - 获取播放链接（必需）
- `lyric` - 获取歌词
- `pic` - 获取封面
- `search` - 搜索音乐（需要新增支持）
